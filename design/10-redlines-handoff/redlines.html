Redlines & specs

Le manuel à ouvrir quand on code un écran. Mesures, tokens appliqués, règles d'usage par primitive, exceptions. La règle d'or : un dev ne devine jamais — soit la valeur est ici, soit elle est dans shared/tokens.css ou packages/design-tokens.

Conventions de cette pageLes annotations rouges (#C4622D) sont des mesures. Les chiffres en mono sont des tokens. Les --s-N renvoient à l'échelle d'espacement (4-base). Toute valeur en pixels indiquée est appliquée — vous lisez le DOM rendu, pas une approximation.
01 · ESPACEMENT

Échelle 4-based · 12 paliers

Tout l'espacement vient de cette échelle. Aucune valeur libre dans le code (revue refusée). Les paliers sont conçus pour produire des rythmes harmoniques (4 → 8 → 16 → 32 = doublements; 12, 20, 24, 56 = paliers tactiques entre les doublements).

✓ Faire

  • Toujours utiliser un token: padding: var(--s-5)
  • Doubler à mesure qu'on monte dans la hiérarchie : carte 20px → section 40px → page 80px

✗ Ne pas faire

  • Valeur libre : padding: 18px
  • Multiples non-tokens (calc) : padding: calc(var(--s-4) + 3px)
  • Inventer un palier intermédiaire — si vous en avez besoin, ouvrez une PR sur shared/tokens.css

08 · NOTES D'USAGE PAR PRIMITIVE

Quand l'utiliser, quand ne pas

Pour les cas non triviaux. Lire avant de proposer une nouvelle variante.

Bouton primaire

✓ Faire

  • Une seule action principale par écran
  • Couleur primary uniquement pour l'action irréversible/positive
  • Texte ≤ 24 ch. Verbe d'action en début (« Confirmer », « Démarrer »)

✗ Ne pas faire

  • Deux primaries côte à côte (utiliser primary + ghost)
  • Primary pour navigation (utiliser ghost ou link)
  • Texte « Cliquer ici » / « OK »

Carte (subscriber)

✓ Faire

  • Eyebrow mono UPPER pour catégoriser (« PROCHAINE VISITE »)
  • Titre Fraunces italique pour le H1 de carte
  • Maximum 4 éléments verticaux : eyebrow + title + body + cta

✗ Ne pas faire

  • Carte cliquable entière sans CTA visible (cible tactile invisible = piège)
  • Plus d'une ombre par carte (jamais d'effet « élévation lourde »)
  • Tordre le border-radius (toujours --r-md)

Tableau (operator)

✓ Faire

  • Hauteur de ligne 36-40px. Plus dense = lisibilité tuée.
  • font-variant-numeric: tabular-nums sur toute colonne chiffrée
  • Sélection clavier J/K + ouverture drawer ↵
  • Header sticky pour les listes > écran

✗ Ne pas faire

  • Cellules plus de 2 lignes (utiliser drawer pour le détail)
  • Tri client-side au-delà de 100 lignes (toujours côté serveur avec query params)
  • Couleur de fond sur ligne pour décorer (réservé à la sélection & aux états critiques)
09 · ACCESSIBILITÉ — CHECKLIST PAR ÉCRAN

WCAG 2.1 AA · 44pt · 200% type

À cocher avant tout ticket « Done ». Aucune exception, aucun trade-off — la barre est mondiale (cf. principe 02).

  • Contraste de texte ≥ 4.5:1 (corps) ou ≥ 3:1 (titres ≥ 24px). Vérifier dans Stark / WebAIM.
  • Cibles tactiles ≥ 44 × 44 pt. Les zones plus petites doivent avoir une zone cliquable étendue (::before avec inset: -10px).
  • Focus visible sur tout élément interactif. box-shadow: var(--focus-ring). Jamais outline: none sans remplacement.
  • Lecteur d'écran — étiquettes ARIA pour icônes seules, ordre DOM logique, aria-live pour les toasts.
  • Type dynamique — texte en rem, pas en px. Vérifier à 200% que rien ne déborde.
  • Couleur seule — un état (succès, erreur) n'est jamais signalé uniquement par couleur. Toujours + icône / + texte.
  • Mouvement — respecter prefers-reduced-motion: reduce. Désactiver les animations > 200ms.
  • Navigation clavier — Tab, Shift-Tab, Esc, ↵ doivent marcher. La palette ⌘K est testée à 100%.
  • Photos — alt descriptif (« Photo avant lavage : panier de linge sale » plutôt que « image »).
  • Formulaires — chaque champ a un <label> ou aria-label. Erreurs liées via aria-describedby.
10 · ERREURS COURANTES — REVUE DESIGN

Les motifs refusés en revue

Ce qui revient en revue à chaque PR. Lisez avant d'ouvrir le vôtre.

Motif Pourquoi refusé Correctif
Couleur littérale dans CSS Empêche le theming par data-theme et casse le thème dark/light futur. Utiliser var(--ink), var(--primary), etc.
Box-shadow custom Brise le rythme d'élévation. 3 niveaux suffisent. Utiliser --shadow-card · --shadow-elevated · --shadow-overlay.
Padding de 18px Hors échelle 4-base. Crée des micro-désalignements. 16 (s-4) ou 20 (s-5).
Police custom (Roboto, Helvetica…) Hors brand. Casse la signature. Fraunces · Inter · JetBrains Mono. Point.
Emoji dans la copie système Voir Voix & ton — interdit (sauf 🆘 SOS). Mots ou icônes SVG.
Bouton « OK » Vide. Refusé. Verbe d'action concret (« Confirmer », « Compris »).
Tutoiement Inadapté en français togolais formel. Vouvoiement systématique.
Animation > 480ms Au-delà, on ressent le décalage. Tout bloque. 120 / 200 / 320 / 480ms — pas plus.
État loading non documenté L'utilisateur croit que ça a marché. Toujours fournir loading + error + empty (cf. 08-states-library).
Texte sur image sans dégradé Lisibilité aléatoire selon la photo. Dégradé sombre 0 → 60% noir alpha .5 par-dessus la photo.

3. Audit accessibilité Lighthouse ≥ 95
