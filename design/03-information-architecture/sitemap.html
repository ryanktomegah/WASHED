01 · MODÈLES DE NAVIGATION

Trois apps, trois patterns

Chaque application a son modèle de navigation propre — calé sur les conventions de sa plateforme et sur le profil de son utilisateur.

Subscriber · iOS (Tab Bar)

  • 4 onglets tab : Accueil · Visites · Abonnement · Profil
  • Pile de navigation par onglet (push/pop · transition slide horizontale)
  • Modales en sheet remontante modal : paiement, signalement, support
  • Pas de hamburger menu — toutes les sections principales sont en tab
  • Deep links deep : push notification ouvre l'écran cible directement
  • Authentification first-run : flux X-01 → X-08, sortie impossible avant complétion

Worker · Android (Tab Bar bas + SOS flottant)

  • 4 onglets tab : Tournée · Gains · Profil · Aide
  • Bouton SOS flottant rouge persistant sur tous les écrans (1 tap = appel direct bureau)
  • Pas de back-stack profond — chaque action revient à la tournée du jour
  • Modales en sheet remontante modal : retrait, signalement, congé
  • Deep links deep : push assignation ouvre détail visite
  • Hors-ligne offline-first : tous les écrans fonctionnent sans réseau

Operator · Web desktop (Sidebar + drawer)

  • Sidebar fixe gauche · 4 sections : Aujourd'hui · Personnes · Finances · Org
  • Vue principale centrale + drawer contextuel droit pour le détail
  • Palette de commandes ⌘K ouvre cherche + actions partout
  • URL routables — chaque vue a une URL partageable copiable
  • Deep links deep : email/Slack ouvre directement le ticket ou la demande
  • Authentification 2FA : par défaut · session 8h max

Conventions partagées

  • Toutes les apps : barre de statut système, jamais cachée
  • Toutes les apps : bouton retour retourne, ne ferme pas l'app
  • Toutes les apps : modale fermable par swipe-down (mobile) ou Esc/click-outside (web)
  • Toutes les apps : URL copiable pour chaque écran principal (mobile = deeplink scheme)
  • Aucune navigation par geste exclusif (toujours équivalent bouton — cf. accessibilité)
