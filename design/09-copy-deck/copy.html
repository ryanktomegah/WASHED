Copy deck — français

Toutes les chaînes des trois applications, classées par app et surface. Chaque entrée a une clé i18n, un contexte d'usage, une limite de caractères, et le texte FR validé. Vocabulaire approuvé, vocabulaire interdit. Source de vérité pour packages/i18n/fr.json.

Comment lire ce documentLes clés i18n suivent un schéma scope.surface.element. Exemple : subscriber.dashboard.next_visit_label. Les variables sont en {accolades}. Pluralisation gérée via ICU MessageFormat. Limites de caractères = visuel + accessibilité (200% dynamic type). Si vous dépassez, le design refuse la chaîne.
00 · VOIX & TON

Le ton Washed en français

Le « vous » est non-négociable. Le tutoiement n'est jamais utilisé, même dans les apps internes — l'usage du « tu » envers une laveuse en français togolais est connoté de manière inappropriée. Le ton est respectueux, factuel, sans excuses inutiles. Un message Washed dit ce qui se passe et ce qu'il faut faire.

✓ Faire

  • Employer le « vous » de politesse partout
  • Phrases courtes (≤14 mots)
  • Verbes d'action concrets (« confirmer », « envoyer », « démarrer »)
  • Heures au format 24h sans zéros : 9 h 00
  • Devises avec espace : 2 500 XOF
  • Italique pour les chiffres-fierté (gain, semaine, mois)
  • Premier prénom + initiale du nom de famille (« Akouvi K. »)
« Mardi 9 h 00 — Akouvi K. arrive chez vous. »

✗ Ne pas faire

  • Tutoyer, jamais
  • S'excuser inutilement (« Désolé, mais… »)
  • Employer du jargon SaaS (« utilisateur », « onboarding »)
  • Anglicismes (« cool », « ok », « shopper »)
  • Emoji dans la copie système (sauf chiffres unicode pour SOS)
  • Formules paternalistes (« Pas de panique ! »)
  • Capitales criantes (« ATTENTION », « URGENT »)
« Désolé, ton paiement a échoué. Pas de panique, ressaye ! »

Format de livraison

Cette page existe en deux formats:

  • copy.html — ce document, source de vérité humaine. Prime sur tout.

Règle d'orToute chaîne en production doit avoir une clé dans ce deck. Si un dev a besoin d'une chaîne qui n'existe pas ici, il l'ajoute en PR à ce document AVANT d'écrire le code. Pas l'inverse.
