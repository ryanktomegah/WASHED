L'accessibilité n'est pas optionnelle.

WCAG 2.1 AA est le plancher, pas le plafond. Cette page documente les contrats que chaque écran doit honorer. Si une décision de design rend l'application inutilisable pour un utilisateur — daltonien, malvoyant, motrice limitée, technologie d'assistance — la décision est mauvaise et doit être révisée.

01 · CONTRASTES

Ratios mesurés

Texte normal ≥ 4.5:1, texte large (≥18pt ou ≥14pt bold) ≥ 3:1, éléments graphiques ≥ 3:1. Chaque combinaison de couleurs est mesurée.

02 · CIBLES TACTILES

44 × 44 pt minimum

Toute zone tappable, sur n'importe quel écran, fait au moins 44 pt sur iOS et 48 dp sur Android. Le visuel peut être plus petit, mais la zone d'interaction respecte le minimum.

04 · CHECKLIST PAR ÉCRAN

Avant de livrer un écran

Un écran qui n'a pas passé cette checklist n'est pas livrable. Le designer le vérifie, le développeur le re-vérifie, le QA le contrôle.

Visuel & couleur

  • ✓
    Tous les contrastes texte/fond ≥ 4.5:1 (≥ 3:1 si grand)contraste.tester
  • ✓
    Aucune information portée par la couleur seule (toujours doublée par texte ou icône)no.color.only
  • ✓
    Mode sombre OS respecté quand l'utilisateur l'active (v2)prefers-color-scheme
  • ✓
    État focus visible sur tout élément interactif (anneau 3px primary)focus-visible

Tactile & moteur

  • ✓
    Toute cible tactile ≥ 44 pt iOS / 48 dp Android, même si le glyphe est petitmin.tap-target
  • ✓
    Aucune action critique uniquement par geste (toujours un bouton équivalent)no.gesture-only
  • ✓
    Confirmation pour toute action destructive (annuler, supprimer, désabonner)destructive.confirm
  • ✓
    Aucune limite de temps pour compléter une saisieno.time-limit

Lecture d'écran

  • ✓
    Tous les éléments interactifs ont un label accessible en françaisaccessibilityLabel
  • ✓
    Ordre de lecture cohérent avec l'ordre visuelreading.order
  • ✓
    Annonces de changement d'état (loading, error, success) via live regionaria-live
  • ✓
    Images de profil ont alt avec le nom; images décoratives ont alt videimg.alt

Texte & saisie

  • ✓
    Texte agrandi jusqu'à 200 % sans débordement ni superpositiondynamic.type
  • ✓
    Champs de saisie ont label persistant (pas placeholder seul)label.persistent
  • ✓
    Erreurs de saisie expliquées en mots, pas codeserror.human
  • ✓
    Auto-complete et inputmode corrects (tel, numeric, otp)inputmode
05 · LITTÉRATIE

Bas niveau de lecture

Une partie du public laveuse a un niveau de lecture limité. La copie suit la règle CE2 — phrases courtes, mots usuels, pas de jargon.

Pour la laveuse

  • ✓
    Phrases ≤ 12 mots
  • ✓
    Verbes à l'impératif présent (« Prenez la photo »)
  • ✓
    Toujours une icône à côté de chaque action critique
  • ✓
    Audio FR optionnel pour les instructions clé (v2)
  • ✓
    Numéros affichés en gros, séparés par groupes de deux

Pour l'abonné

  • ✓
    Phrases ≤ 18 mots, niveau presse quotidienne
  • ✓
    Vocabulaire courant — pas de termes financiers ou techniques sans glose
  • ✓
    Toute mention légale a un résumé en une phrase au-dessus
  • ✓
    Anglais différé : pas avant le marché Accra/Ghana en v3
